Цей код представляє систему для надсилання повідомлень
із використанням паттерну "Фасад" (Facade pattern). 
Дозвольте розглянути кожен клас окремо та пояснити їх функціонал:

    Logger class (Клас Logger):
        Відповідає за логування повідомлень в консоль.
        Має метод log, який приймає рядок message та виводить 
        його в консоль.

    Notifier class (Клас Notifier):
        Відповідає за відправлення повідомлень.
        Має метод send, який приймає шаблон template та 
        адресата to та виводить інформацію про відправку в консоль.

    TemplateManager class (Клас TemplateManager):
        Утримує список шаблонів у приватному полі #templates.
        Має метод getByName, який приймає ім'я шаблону (name) 
        і повертає відповідний шаблон або undefined, якщо шаблон 
        не знайдено.

    NotificationFacade class (Клас NotificationFacade):
        Об'єднує логіку логування, відправлення повідомлень та 
        управління шаблонами в одному місці.
        Має приватні поля для логгера (#logger), нотіфікатора 
        (#notifier) 
        та менеджера шаблонів (#templateManager).
        Має метод send, який викликає getByName для отримання 
        шаблону за ім'ям і, 
        якщо шаблон існує, відправляє його за допомогою нотіфікатора 
        та логує подію.

    User class (Клас User):
        Представляє користувача із збереженим емейлом та інтерфейсом 
        для оновлення інформації.
        Має приватне поле для емейла (#email) та об'єкт 
        NotificationFacade 
        для взаємодії із системою нотифікацій.
        Має метод updateInfo, який представляє бізнес-логіку оновлення 
        інформації 
        та викликає send з NotificationFacade для надсилання 
        повідомлення про оновлення.

    Order class (Клас Order):
        Представляє замовлення із збереженим емейлом та інтерфейсом 
        для скасування замовлення.
        Має приватне поле для емейла замовника (#customerEmail) та 
        об'єкт NotificationFacade 
        для взаємодії із системою нотифікацій.
        Має метод cancel, який представляє бізнес-логіку скасування 
        замовлення 
        та викликає send з NotificationFacade для надсилання 
        повідомлення про скасування.

Ця структура дозволяє ізолювати логіку нотифікацій від різних 
частин системи, спрощуючи їх 
взаємодію і підтримку. Фасад (NotificationFacade) виступає як 
одна точка доступу для 
нотифікаційної системи для інших частин програми.
